ffmpegGUI is a graphical interface to help constructing the commandline for ffmpeg to encode/convert video files. You can also encode files directly with ffmpegGUI and avoid having to open Terminal to paste the assembled ffmpeg commandline. You can also create encoding jobs to process them all in a batch later.

You start by selecting the source file, either via drag and drop, a click on Source file, or by opening a video directly with ffmpegGUI from its Open with… context menu in Tracker. Below the file path, you'll see media info about the source: codecs, resolution, framerate, bitrates, duration etc.

Choosing the source automatically sets the output file to the same name with the container format's extension, by default ".avi". You can change the output filename and format, of course.
In the Options below, the video bitrate, framerate and resolution and audio bitrate, sampling rate and channel count also get automatically filled with the source file's properties. They are a good basis for the changes you may want to make before converting.

The play buttons "⯈" will open the source - and once created, the output - in your preferred media player.

Once you've configured all options (see below), you either click Start to begin encoding, or choose Add as new job from the Jobs menu (see Job manager below).

While encoding, the Start button becomes Stop, letting you abort the encoding process. Activate the checkbox Play when finished to the right of progress bar as an additional finishing notification.

Options

Deactivating the checkmark Enable video / audio encoding allows you to completely remove the video or audio track from the encoding.

The Video / Audio codec menu offers all available codecs. Choosing 1:1 copy will use the same settings as the input file. This is also called "transcoding", as this saves the very CPU intensive and time consuming process of encoding, as it just makes a perfect copy of the track.

You can change Bitrate (Kbit/s) and Framerate (fps) and activate the checkbox to Use custom resolution.

The rest of the options for the audio track work similarly.

Every change you make is immediately reflected in the ffmpeg commandline below the options. If, for example, you need an audio bitrate that's not available from the menu, you can look for the corresponding parameter in the commandline and manually change the value.

Note, however, that any further change you make in the options will re-create the commandline and overwrite your manual additions. Manual tweaking should therefore always be the last step before encoding!

Cropping

The tab Cropping is only active if you actually process the video track, so you won't be able to enter any cropping values while the video codec is 1:1 copy. Cropping shows a frame of the source video:

New preview fetches another random frame of the video if the one currently displayed isn't suitable.

You can set the amount of pixels to cut off from the Top, Bottom, Left and Right. An overlayed box illustrates how the picture will be cropped.

Reset zeroes the coordinates.

Log

You'll find the output of the ffmpeg command in this Log tab. Mostly, it's just the progress output that's also shown as a bar at the bottom of the window. But if something should go wrong, you'll find the error message here as well.

Job manager

If you have not just a single video to process, you can opt to configure each encoding first and queue it in a list of jobs, instead of encoding every file separately by clicking Start and waiting for it to finish.

To create a job from the current settings, you choose Add as new job from the Jobs menu. The same menu has an item to Open job manager… which opens this window:

Start all jobs will begin to encode all jobs in the list, from top to bottom. Similar to the main window, it'll change to Abort all jobs when the encoding is in progress.

You can change the order of the jobs by sorting the columns, or move a single job up or down by selecting it and clicking the ⏶ or ⏷ buttons at the bottom.

Remove deletes the currently selected job and Error log shows its error output (if something went wrong).
Clear finished removes all successfully encoded jobs.

You can also just encode the currently selected job, easiest with a right-click on a job and selecting Start this job.

Double-clicking a single job will do the intuitive thing, depending on its status: If it's Waiting it'll start encoding it, if it's Running it aborts it, if it's Error it opens its log, if it's Finished it plays back the output file.

You can send a job back to the main window to change its settings by selecting a job and choosing Edit this job from the context menu. That will remove the job from the job manager. Once you're done with tweaking the options in the main window, just do an Add as new job there, and it's back in the list of jobs.

ffmpegGUI will save all unfinished jobs when it's quit, so you can continue when you're back.

Download

ffmpegGUI is directly available through HaikuDepot from the HaikuPorts repository. You can also build it yourself using Haikuporter. The source is hosted at GitHub.

Bugreports & Feedback
